## Reflection (100–150 words)
Through this challenge, I have gained a deeper understanding of version control as a foundational tool for modern software development. I learned that Git is not just about saving files; it's about tracking the evolution of a project and enabling seamless collaboration. The most difficult step was managing the transition between the local environment and the remote GitHub repository, specifically ensuring that branches were correctly pushed and tracked. 

Branches and pull requests are incredibly useful because they provide a structured way to review changes before they are integrated. This prevents bugs from reaching the main codebase and allows for team discussion and feedback. Overall, this process has shown me how professional developers maintain code quality and manage complex projects efficiently.
